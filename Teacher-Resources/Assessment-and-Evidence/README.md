**Assessment & Evidence in Minecraft Coding**<br>
This folder contains teacher‑facing resources designed to support valid, practical, and sustainable assessment of student coding work in Minecraft Education using MakeCode Python.
These documents address different—but related—questions teachers commonly have when teaching coding in Minecraft. Each resource has a distinct purpose, and they are intended to be used together, not interchangeably.

📄 **Evidence of Success in Minecraft Coding**<br>
What this document is for
Defines what counts as valid evidence that student code executed successfully in Minecraft Education.
This document reassures teachers that visible world changes and outcomes—such as messages, spawned entities, or environmental changes—are legitimate forms of assessment evidence, even when traditional line‑by‑line code inspection is limited or unnecessary.
Use this document if you are asking:

“How do I know that a student’s code actually worked?”
“Does observing the world really count as evidence?”

This document explains:
What counts as evidence of success.
This document does not do:
Explain how to collect evidence or prescribe assessment workflows.

📄 **Assessment Progression: Evidence Collection Strategies**<br>
What this document is for
Describes how methods for collecting and reviewing assessment evidence evolve over time as students and teachers gain technical skills and confidence.
This document helps teachers move from early, observation‑based assessment toward efficient, repeatable workflows, such as reviewing shared code links, without increasing grading burden.
Use this document if you are asking:

“What should assessment look like early in the course versus later?”
“When can I rely on links instead of watching every student run code?”

This document explains:
How evidence collection practices develop and stabilize across lessons.
This document does not do:
Justify why Minecraft outcomes count as evidence or provide teaching strategies.

📄 **Formative Assessment Strategies in Minecraft Education**<br>
What this document is for
Provides practical classroom guidance for teaching and assessing coding in Minecraft Education.
This resource focuses on teacher moves, instructional norms, and common classroom scenarios, drawing on real teaching experience rather than prescribed progressions.
Use this document if you are asking:

“What should I do in this situation?”
“What works well with students when teaching or assessing in Minecraft?”

This document explains:
Instructional and assessment strategies that support student learning.
This document does not do:
Define assessment standards or formal evidence requirements.

***How These Documents Work Together***

Evidence of Success establishes what counts
Assessment Progression explains how assessment practices grow
Formative Strategies supports what teachers do day‑to‑day

Taken together, these resources aim to support teacher confidence, clarity, and sustainability when assessing coding learning in Minecraft Education.
