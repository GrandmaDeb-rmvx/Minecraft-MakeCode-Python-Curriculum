
# U01L03 – Tell a Story
Overview
In this lesson, students use the player.say() command to create a short, Minecraft-themed story, conversation, or message. The focus is on using sequencing and clarity to communicate meaning through code rather than speed or complexity.
Estimated Time: 45 minutes

**Focus Skills**

Using a command repeatedly with different arguments
Sequencing instructions so output appears in a meaningful order
Writing clear, connected lines of dialogue or narration
Applying creativity within structured constraints
Following basic formatting and commenting conventions in code


**Evidence of Completion**
Student work should demonstrate:

A new MakeCode Python project
Six or more properly formatted player.say() commands
Lines that:

Make sense individually
Connect to a single topic or idea


Minecraft-related content (characters, locations, actions, or themes)
Working code that runs without errors
A shared project link submitted as instructed


**Success Criteria (At a Glance)**<br>
✅ New project started<br>
✅ At least 6 player.say() lines<br>
✅ Commands run in the intended order<br>
✅ Message or story is clear and cohesive<br>
